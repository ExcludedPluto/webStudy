package.json을 보면 이미 test에 관한 명령어들을 미리 설정해 놓음

각 test 명령어들은 .spec.ts 파일을 보고 테스트를 함.

따라서 어떠한 파일을 테스트하고 싶으면 그 파일의 .spec.ts 파일이 있어야함




test:cov

	어떤 파일이 테스트 됐고, 얼마나 테스트 됐는지, 어떤 코드가 테스트 안됐는지 알려줌


test:watch
	
	모든 .spec.ts 파일을 찾아내서 테스트를 실행하거나 테스트를 watch 하고 있는 것
	watch 하고 잇는 테스트 파일이 변경될때마다 자동으로 테스트를 진행한다.


test:e2e
	
	end to end 테스트를 할 때 사용함

	유닛 테스트 : 함수 하나를 따로 테스트 하는 것
	e2e 테스트 : 사용자의 관점에서 서비스를 이용할때 과정을 따라가며 테스트하는 것.
